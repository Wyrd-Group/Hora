LibraryLicensePrioritySpec FeatureWhy
Colyseus MIT ESSENTIAL Real-time multiplayer sync — Syndicate Holding Companies, live PvP Hostile Takeovers, M&A Cyber-Strikes. Room-based architecture = one room per "game world" or "syndicate battle." Node.js. Automatic binary delta state sync (only sends changes). Server-authoritative by design. Matchmaking built-in. Horizontal scaling. Perfect thin-client model — browser receives state snapshots, renders them, sends actions back.
Celery + Redis broker BSD-3 ESSENTIAL Construction timers (Build vs. Buy mechanic), quarterly economic ticks, AI Director event generation, gacha pack pulls, board vote scheduling, Heat decay (−1/day), salary/tax calculations. Durable workflows that survive server restarts. An M&A battle can span days with multiple player inputs — Temporal tracks state across the entire saga. Overkill for simple timers (use Celery), essential for complex multi-step game events.
Valkey BSD ESSENTIAL Real-time state cache (wallet balances, corporation status), pub/sub for event broadcasts (crime busts, market crashes), leaderboard sorted sets, session management. Open-source Redis fork (Linux Foundation). Drop-in Redis replacement. Sub-millisecond reads. Your Celery broker runs on this too. One dependency, multiple uses.
bitECS MIT RECOMMENDED Entity Component System for managing all game entities server-side: corporate nodes, AI rival corporations, trade routes, departments, employees/manager cards, venues. Data-oriented design. Extremely fast iteration over thousands of entities. Composable systems (TaxSystem, CrimeSystem, IncomeSystem run independently). Natural fit for the 12 venues × 5 levels × 6 departments × N players entity space.
Architecture Note: Colyseus (Node.js) handles real-time multiplayer sync. FastAPI (Python) handles REST endpoints + webhook actions. They communicate via Valkey pub/sub. This is intentional — Colyseus is best-in-class for WebSocket state sync, FastAPI is your existing backend with all your Python engines. Don't collapse them into one.

PART 6: HARD TRUTHS & RISK ASSESSMENT

What will actually be hard

  1. Colyseus ↔ FastAPI bridge: Two different runtimes (Node.js + Python) talking via Valkey pub/sub. This is architecturally sound but operationally complex. Expect 2 weeks just on the integration layer in Phase 3.
  2. Deck.gl custom shaders for "glow" effect: The additive blending trick works, but getting the exact Palantir/DEFCON aesthetic requires custom WebGL shader work. Budget time for visual polish — it's what sells the game to investors.
  3. AI Director quality: LangGraph gives you the orchestration. But the quality of procedural events depends on prompt engineering and game design. Garbage in, garbage out. This is a game design problem more than a tech problem.
  4. Scale of the spec vs. your timeline: The full spec is 2-3 engineer-years of work. Phase 1 alone is an impressive product. Don't let scope creep kill you before the October beta. Ship Phase 1, get users, then iterate.
  5. Temporal operational overhead: Running a Temporal cluster adds infrastructure complexity. For the first pass, you can model M&A battles as Celery task chains. Move to Temporal only when you need multi-day durable workflows.

What you should NOT build

  1. Don't build your own map renderer. MapLibre + Deck.gl gives you everything. Custom Canvas/WebGL map rendering would take months and be worse.
  2. Don't build your own multiplayer sync. Colyseus handles binary delta state sync, reconnection, room management. You'd spend 3 months reimplementing what it does in 3 days.
  3. Don't build your own charting library. ECharts has 40+ chart types with dark theme. D3 for custom one-offs only.
  4. Don't use Kafka. RedPanda if you need event streaming. Kafka's JVM + Zookeeper overhead is absurd for a pre-revenue game.
  5. Don't try to make CesiumJS work for the main view. 3D globe is heavier, harder to customize, and slower to iterate on. Flat tactical map first. Globe.gl for endgame "Emperor mode" only.

What gives you the biggest bang for the buck

  1. MapLibre + Deck.gl + CARTO Dark: Instantly transforms the game from "student project with 3D cubes" to "holy shit is this Palantir?" The aesthetic is 80% of the pitch.
  2. cmdk + shadcn/ui terminal: The Data Intel Feed is the game. It's how players interact with everything. Nail this and the game feels like a real command interface.
  3. Your existing 19 engines: Already built, already tested. The open-source stack just gives them a delivery mechanism. Your engines are the moat. Everything else is plumbing.
