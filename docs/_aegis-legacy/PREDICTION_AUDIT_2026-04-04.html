1. Executive Summary

Bottom line: The Monte Carlo log-normal model is a marginally useful directional signal at aggregate level (53.1% accuracy), but becomes actionable when the predicted move exceeds 3% (55.9% accuracy). The 10/90 confidence bands are slightly too narrow overall (75.9% vs 80% target), meaning the model underestimates tail risk — particularly for high-beta names like AAPL, COIN, PLTR, and SHOP.


Key finding: The model performs best on MSFT (76% directional accuracy), GOOGL/UBER/COIN (64%), and TSLA (60%). It actively fails on AMD (28%), AMZN/NVDA/AAPL/META (44%). This is not random — the model systematically extrapolates recent drift, so it works on assets with trending behavior and fails on mean-reverting or news-driven ones.

7. Honest Assessment & Recommendations

What's Broken

1. FinBERT sentiment is returning empty results. NewsAPI key may be exhausted or expired. Without sentiment overlay, the model is running blind on news-driven moves — which is exactly where the MC model fails worst (AAPL, META, AMZN all at 44% accuracy).

2. Band calibration is too tight (75.9% vs 80% target). The model uses historical sigma as-is. It should apply a volatility scaling factor of ~1.15x to widen bands. AAPL is the worst offender at 56% band coverage — the model is systematically overconfident on AAPL.

3. AMD is adversarial to this model (28% directional accuracy). That's worse than a coin flip — the model is actively anti-predictive for AMD. It should be excluded from the signal set or given an inverse weight.

What's Working

1. MSFT is the model's best asset (76% accuracy). MSFT trends cleanly and the drift model captures it well.

2. The 3% edge threshold is real. At >= 3% predicted move, win rate jumps to 55.9% over 186 forecasts. That's a statistically meaningful signal — enough to generate alpha with proper position sizing.

3. Crypto band coverage is well-calibrated (BTC 77.8%, ETH 84.4%) — the model correctly accounts for crypto's higher volatility.

Concrete Next Steps (Priority Order)

1. Fix NewsAPI integration — Verify API key, check quota. Sentiment overlay is the single biggest accuracy multiplier you're missing. A forecast + bearish FinBERT score has much higher conviction than forecast alone.

2. Apply volatility scaling (1.15x sigma) — One-line change in monte_carlo_forecast(). Will fix the band under-coverage problem across the board.

3. Implement per-asset accuracy weighting — Don't trust all forecasts equally. MSFT at 76% accuracy should get full Kelly sizing. AMD at 28% should get zero. Weight by historical win rate.

4. Add regime filter — VIX at 24.5 means elevated vol. The MC model uses trailing sigma which underestimates vol during regime shifts. When VIX > 25, scale sigma by 1.3x.

5. Run sports scanner on event-day schedules — The value is in pre-match windows, not 24/7 polling.
