A. Entity Model Adaptation (Aegis → Empire)

Current: Intelligence Entities

  id, uid, callsign, entityType: 'aircraft'|'vessel'|'ground'|'event'|'person',
  affiliation: 'friendly'|'hostile'|'neutral'|'unknown',
  lat, lon, altitude, heading, speed, source, lastSeen
}

Needed: Corporate Game Nodes

EmpireNode {
  id, name, nodeType: 'venue'|'department'|'industrial'|'shell'|'rival',
  owner: 'player'|'market'|'rival_ai_id',         // Red vs Green vs Enemy
  sector: 'oil_gas'|'tech'|'manufacturing'|'energy'|'finance'|'pharma',
  lat, lon,
  level: 1-5,                                       // Venue upgrade level
  capex, opex, monthlyIncome, throughputMultiplier,
  cardSlots: { manager: Card|null, specialist: Card|null, security: Card|null },
  constructionStatus: 'operational'|'building',
  constructionEndsAt: timestamp|null,
  heatContribution: number,                          // Crime heat from this node
  esgScore: number,
}

The adaptation: Extend mssStore.ts with a parallel empireStore.ts for game state. The existing entity system stays for Analysis Mode (rival espionage). Empire nodes are a new Deck.gl layer on the same MapViewer — just add a new ScatterplotLayer for corporate nodes alongside the existing intelligence layers. The toggle system already supports this pattern.
