API ENDPOINTS (35+ total)
================================================================================

Health & Status:
  GET /                    - Health check
  GET /api/status          - Engine availability

Portfolio:
  POST /api/portfolio/optimize      - Optimize portfolio
  GET /api/portfolio/returns        - Historical returns

Forecasting:
  POST /api/forecast/{symbol}       - Forecast single symbol
  POST /api/forecast/multi          - Forecast multiple symbols

Market Data:
  GET /api/market/ohlcv/{symbol}    - OHLCV data

Anomaly Detection:
  POST /api/anomaly/scan            - Scan for anomalies
  POST /api/anomaly/analyze         - Analyze entities

Threat Analysis:
  POST /api/threat/rank             - Rank by threat score

Clustering:
  POST /api/clustering/dbscan       - DBSCAN clustering

Link Analysis:
  POST /api/links/graph             - Build entity graph

Kalman Filtering:
  POST /api/kalman/track            - Smooth and predict tracks

Explainability:
  POST /api/explain/threat          - Explain threat scores

Market Intelligence:
  GET /api/predictions/scan         - Scan prediction markets
  GET /api/sports/scores/{league}   - Live sports scores

Sentinel:
  GET /api/sentinel/status          - Sentinel status
  POST /api/sentinel/config         - Update configuration
  GET /api/sentinel/alerts          - Get recent alerts

================================================================================
ENGINE INTEGRATIONS
================================================================================

All 11 Quadratic engines are integrated with graceful error handling:

1. portfolio        - Portfolio optimization (riskfolio-lib)
2. forecasting      - Time-series forecasting (bolt models)
3. drl_training     - Deep RL training (PPO, DQN)
4. anomaly          - Anomaly detection (statistical, behavioral)
5. kalman           - Kalman filtering (tracking, prediction)
6. clustering       - Clustering (DBSCAN, hierarchical)
7. threat           - Threat scoring and ranking
8. link_analysis    - Entity relationship graphs
9. explain          - Threat score explainability
10. market_intel    - Market intelligence & prediction markets
11. intelligence    - Intelligence analysis

Data Feeds:
- market_data      - Stock/crypto prices, OHLCV
- ais_data         - Vessel tracking, scenarios
- ml_benchmarks    - Classification/regression datasets

If an engine fails to import, the API continues running with that endpoint
returning 503 with helpful error message. All features degrade gracefully.

================================================================================
CONFIGURATION
================================================================================

Environment Variables:
  PORT=8888              - Server port (default: 8888)
  LOG_LEVEL=INFO         - Logging level (default: INFO)
  
  Additional options in .env.example

CORS:
  Currently configured for localhost only
  Update allow_origins in server.py for production

Response Format:
  All endpoints return consistent JSON wrapper:
  {
    "status": "ok|error",
    "data": {...},
    "engine": "engine_name",
    "timestamp": "ISO-8601"
  }

DEPLOYMENT OPTIONS
================================================================================

1. DEVELOPMENT (local testing)
   python server.py

2. PRODUCTION - DOCKER (recommended)
   docker build -t quadratic-api:1.0.0 .
   docker run -p 8888:8888 quadratic-api:1.0.0

3. PRODUCTION - GUNICORN (traditional)
   gunicorn -w 4 -k uvicorn.workers.UvicornWorker api.server:app

4. PRODUCTION - SYSTEMD (Linux)
   Create service file and: systemctl start quadratic-api

See DEPLOYMENT_CHECKLIST.md for complete procedures.
